The Juno.jl Front-End

Enhanced Display

Juno.jl includes features which allow package developers to created enhanced displays in Juno. For example, we can print info to the console in a nicer way:

info("foo")

The following functions provide this interface:

Media.media — Function.

media(T) gives the media type of the type T. The default is Textual.

media(Gadfly.Plot) == Media.Plot
source
Media.@media — Macro.

Similar to abstract:

@media Foo

defines Foo, as well as FooT, the type representing Foo and its descendants (which is useful for dispatch).

@media Bar <: Foo
Bar::FooT
source
Media.setdisplay — Function.
setdisplay([input], T, output)

Display T objects using output when produced by input.

T is an object type or media type, e.g. Gadfly.Plot or Media.Graphical.

display(Editor(), Image, Console())
source
Media.getdisplay — Function.
getdisplay(T)

Find out what output device T will display on.

source
Juno.info — Function.
info(msg)

Show the given message in Juno's console using blue styling, or fall back to Base.info.

In a package, you can use import Juno: info to replace the default version with this one.

source

Progress Meters

Juno.jl allows package developers to use the progressbar which is provided in the Atom window. For example, you can easily show a progress meter for a for loop via the command:

@progress for ...

The following functions provide this interface:

Juno.@progress — Macro.
@progress for i = ...

Show a progress metre for the given loop if possible.

source

Console Interactions

Juno.jl lets package developers interact with users via the Juno console. For example, you can allow the user to select from a list of options:

selector(xs) -> x

or get input in the console:

input()

The following functions provide this interface:

Juno.isactive — Function.
isactive()

Will return true when the current Julia process is connected to a running Juno frontend.

source
Juno.input — Function.
input() -> "..."

Prompt the user to input some text, and return it.

source
Juno.selector — Function.
selector([xs...]) -> x

Allow the user to select one of the xs.

xs should be an iterator of strings. Currently there is no fallback in other environments.

source
